# Build Configuration Audit

## Project Root

- **Location**: Directory containing `next.config.ts`, `package.json`, `src/`, and `prisma/`
- **Isolation**: `outputFileTracingRoot` ensures Next.js uses only this directory
- **Parent lockfiles**: Ignored; project-local `package-lock.json` and `node_modules` are used

## Configuration

| File | Purpose |
|------|---------|
| `next.config.ts` | outputFileTracingRoot = project root; images (remotePatterns) |
| `tsconfig.json` | paths @/* → ./src/*; strict mode; includes .next/types |
| `prisma/schema.prisma` | Client output → node_modules/@prisma/client |
| `package.json` | build: prisma generate && next build; postinstall: prisma generate |

## Prisma Paths

- Schema: `prisma/schema.prisma`
- Client: `node_modules/@prisma/client` (generated by `prisma generate`)
- Next.js opts Prisma out of bundling automatically

## Vercel

- Uses project root as build context
- No `vercel.json` required
- Env vars configured in Vercel dashboard
